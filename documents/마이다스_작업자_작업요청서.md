# 홈페이지 구조화 데이터 & robots.txt 적용 요청서

마이다스커뮤니케이션 홈페이지 작업자께 전달하는 작업 요청서입니다.

---

## 1. 목적 (한 줄)

검색엔진과 AI(ChatGPT·Gemini·Perplexity 등)가 회사 정보를 정확히 인식하도록, 각 페이지에 **구조화 데이터(스키마, JSON-LD)** 와 **robots.txt** 를 적용하는 작업입니다. 화면에는 보이지 않는 메타데이터이며, 디자인에는 영향을 주지 않습니다.

## 2. 함께 전달하는 파일

- **마이다스_스키마_robots_적용가이드.md** — 실제로 넣을 코드(JSON-LD 블록)와 robots.txt 내용이 모두 들어 있습니다. 페이지별 삽입 위치 표(9번)도 그 안에 있습니다. 이 요청서는 "무엇을 할지" 목록이고, 실제 코드는 그 파일에 있습니다.

## 3. 작업 항목 (체크리스트)

1. ☐ 각 페이지 `<head>`에 해당 JSON-LD 블록 삽입 — 가이드 9번 **‘페이지별 삽입 위치’ 표**대로 넣어주세요.
2. ☐ **스테이징(github.io) 단계 동안** 모든 페이지 `<head>`에 아래 한 줄 추가 (임시 사이트가 검색에 잡히는 것 방지):
   `<meta name="robots" content="noindex, nofollow">`
3. ☐ **실제 도메인(예정: www.midasonair.co.kr)으로 이전한 뒤:**
   - ☐ 위 noindex 줄 삭제
   - ☐ robots.txt(전체 허용)를 도메인 루트(`/robots.txt`)에 배치 — 내용은 가이드 7번
   - ☐ sitemap.xml 생성 후 robots.txt에 적힌 주소(`/sitemap.xml`)에 배치
4. ☐ 모든 URL을 **실제 접속 주소와 동일하게 통일** (https, www 포함). 가이드의 도메인이 실제와 다르면 일괄 치환.
5. ☐ 홈페이지는 가능하면 **루트(`/`)로 서빙** (현재 `midas-communication.html`).
6. ☐ 푸터의 회사 정보(상호·주소·연락처)와 **스키마 내용이 일치**하는지 확인.
7. ☐ 적용 후 검증 — 오류가 없는지 아래 두 곳에서 확인:
   - Google Rich Results Test: https://search.google.com/test/rich-results
   - Schema Markup Validator: https://validator.schema.org

## 4. 꼭 지켜주실 것 (중요)

- **robots.txt에서 검색·AI 크롤러를 막지 말아주세요.** 가이드의 robots.txt는 ‘전체 허용’입니다. 저희는 AI·검색이 사이트를 읽고 추천해주길 원하는 쪽이라, `Disallow`로 막는 설정을 넣으면 안 됩니다. (요즘 AI 봇을 습관적으로 차단하는 경우가 있어 특히 당부드립니다.)
- 비공개로 둬야 할 자료가 있다면 robots가 아니라 **접근 권한(비밀번호 등)** 으로 관리해주세요.
- 스키마 내용은 **화면에 보이는 문구·사실과 어긋나면 안 됩니다** (연혁·주소·인증·서비스명 등 동일하게).

## 5. 일정 안내

- 스키마 블록(JSON-LD)은 **지금 스테이징에 미리 심어둬도 됩니다.** 실제 도메인 기준으로 작성돼 있고, 스테이징은 noindex 상태라 색인되지 않습니다.
- robots.txt(전체 허용) 적용과 noindex 삭제는 **실제 도메인 정식 오픈 시점**에 진행해주세요.
- 정리하면: ① 지금 스테이징 = noindex로 막기 + 스키마 심기 → ② 실제 도메인 이전·SSL 확인 → ③ noindex 삭제 + robots.txt 적용.

## 6. 완료 후 회신 부탁드립니다

- Google Rich Results Test / Schema Validator 통과 결과 (스크린샷이면 더 좋습니다)
- 진행 중 막히는 부분이 있으면 알려주세요. 스키마(JSON-LD)·robots.txt는 표준 웹 기술이라 참고 자료가 많습니다.
